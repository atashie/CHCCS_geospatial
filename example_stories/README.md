# Example Stories

Editorial-style scrollytelling pages that compare specific schools using the same data and methodology as the neutral analysis pages in `src/`.

These are **editorial** (they make arguments based on the data) as opposed to the methodology walkthroughs in `src/` which are **neutral** (they explain how the analysis works).

## Ephesus vs. Seawell Comparison

**File:** `ephesus_seawell_story.py`
**Output:** `ephesus_seawell_comparison.html`

A data-driven comparison of Ephesus Elementary and Seawell Elementary across four dimensions:

1. **Socioeconomic Status** — poverty counts, affordable housing, drive-time vs. attendance-zone demographics
2. **Race/Ethnicity** — dot-density mapping, minority counts, Hispanic population
3. **Age Distribution** — young children (0–4), elementary age (5–9), future enrollment demand
4. **Real Estate Trends** — MLS home sales counts, median prices by zone (2023–2025)

### Color Scheme

- **Ephesus** = red (`#C62828`)
- **Seawell** = blue (`#1565C0`)
- **Combined/actual values** = purple (`#6A1B9A`)
- **Other schools** = gray (`#cccccc`)

### Narrative Structure

The story opens with **methodology first** (steps 0–3): introducing the district, explaining attendance zones vs. drive-time zones, then focusing on Ephesus and Seawell. This ensures readers understand the zone concepts before seeing school comparisons.

All comparisons lead with **counts over percentages** — real human impact matters more than rates when zone populations differ significantly. Seawell's attendance zone is geographically large but sparsely populated in its outer reaches.

**Drive-time zones are the primary lens** since any closure triggers rezoning. Attendance zones are shown as dashed overlays for comparison.

### Key Analytical Insight

Seawell's attendance zone appears economically vulnerable, but drive-time analysis reveals much of that vulnerable population is actually closer to other schools. Meanwhile, Ephesus serves a larger, more diverse population with more young children and nearly 3x the affordable housing units.

### Usage

```bash
# Generate the HTML (requires cached data from prior analyses)

```

### Architecture

Same two-column scrollytelling pattern as `src/socioeconomic_story.py`:
- 45% narrative (left, scrollable) / 55% Leaflet map (right, fixed)
- Scrollama 3.2.0 step triggers
- Leaflet 1.9.3 with CARTO light basemap
- Canvas-rendered dot-density layer (~95K dots)
- Pure HTML/CSS bar charts (no external charting library)
- All data embedded as JS variables in the HTML
- Self-contained single HTML file (typically 5–8 MB)
